# 🌌 The Hitchhiker's Guide to AI Agents

> *"Don't Panic."*
> — The Guide (and also good advice for your first agent deployment)

Far out in the uncharted backwaters of the unfashionable end of the Western Spiral Arm of the internet, there lies a small, largely forgotten collection of GitHub repositories.

This is not one of those repositories.

This is **The Hitchhiker's Guide to AI Agents** — a field guide for the bewildered, the curious, and the developer who just got assigned to "add some AI agents to the product" and has three weeks to figure out what that means.

The AI agent landscape is, to use a technical term, *a lot*. New frameworks appear weekly. Researchers publish papers with names like *"Reflexion: Language Agents with Verbal Reinforcement Learning"* and expect you to have read them. Everyone at every conference is very confident. Nobody agrees on what an agent actually is.

**This guide exists because someone has to be honest about all of that.**

---

## 📖 What This Is

This is not a collection of copy-paste demos (though it has those too).
This is not a list of frameworks (though it covers them).
This is not a tutorial that assumes you already know everything (looking at you, every other tutorial).

This is a **guide**. You read it. It explains things. It has opinions. It will tell you when something is overcomplicated and when the simple version is fine. It will not pretend that all frameworks are equally good for all use cases.

It is, in the tradition of its namesake, *mostly harmless*.

## 🚀 Start Here

If you are new to agents, start with **[01 · Foundations](./01_foundations/)**.

It covers:
- What an AI agent actually is (the useful definition, not the Wikipedia one)
- The three things that make an agent an agent
- How the ReAct loop works
- What's inside an agent: brain, tools, memory, orchestration
- A glossary of every term people will throw at you

If you already know the basics, jump to **[02 · Patterns](./02_patterns/)** for code.

## 🛸 Who This Is For

- Developers who've used LLMs but haven't built agents yet
- Developers who've built agents and want to understand *why* they work (and why they sometimes don't)
- Technical PMs who need to stop nodding and start understanding

## ⚠️ A Word on Frameworks

This guide covers several AI agent frameworks. It has opinions about them. Those opinions are based on their strengths and weaknesses for particular use cases, not on sponsorship arrangements, which is a thing that exists in this space and which you should be aware of when reading other guides.

When the guide recommends something, it says why. When it doesn't recommend something for a particular use case, it says that too.
